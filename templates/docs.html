Quick Start

Clone the repo, install the single dependency, and run:

# 1 — clone
git clone https://github.com/Sp3ct3r137/arctic-nord-site.git
cd arctic-nord-site

# 2 — create a virtual environment (recommended)
python -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate

# 3 — install Flask
pip install -r requirements.txt

# 4 — start the dev server
python app.py

# Open http://localhost:5000 in your browser

The dev server runs with debug=True — Flask will auto-reload whenever you save a Python or template file. Set debug=False before deploying to production.

Routes

All routes are defined in app.py:

GET /

Renders index.html. Passes a features list (Python dicts) into the Jinja2 template context so the feature cards are data-driven — edit the list in app.py to change the cards without touching the HTML.

GET /about

Renders about.html. Passes the full palette list (16 Nord colours + orange). The template uses {{ '{{' }} palette | groupby('group') {{ '}}' }} to automatically sort swatches into their palette groups.

GET /docs

Renders this page (docs.html). Static content only.

API Reference

GET /api/palette

Returns the full colour set as JSON. No auth required.

GET /api/palette

Response 200 OK
Content-Type: application/json

{
  "count": 12,
  "palette": [
    {
      "name":  "nord0",
      "hex":   "#2E3440",
      "group": "Polar Night",
      "role":  "Background"
    },
    ...
  ]
}

You can extend this endpoint — add query params for filtering by group, or wire it to a database if you want dynamic theme management.

Colour Tokens

Every colour lives as a CSS custom property in static/css/style.css under :root. The semantic aliases are what you actually use in component styles: 

/* Raw Nord values */
--nord0: #2E3440;   /* darkest background */
--nord8: #88C0D0;   /* primary frost accent */
--orange: #D08770;  /* CTA / warm accent */

/* Semantic aliases (use these in components) */
--color-bg:      var(--nord0);
--color-surface: var(--nord1);
--color-text:    var(--nord5);
--color-accent:  var(--nord8);   /* frost blue */
--color-cta:     var(--orange);  /* warm orange */

Never hard-code hex values in component CSS — always reference a token. This makes re-theming a one-file change.

Extending the Site

Adding a new page

  • Add a route function in app.py decorated with @app.route("/your-path").
  • Create templates/your-page.html and start it with {% raw %}{% extends "base.html" %}{% endraw %}.
  • Add a nav link in base.html inside .nav__links.

Changing colours

  • Open static/css/style.css and edit the :root block.
  • The semantic aliases (--color-bg, --color-cta, etc.) are the only values your component styles should reference — update the raw Nord vars and everything cascades automatically.

Adding a new API endpoint

@app.route("/api/my-endpoint")
def my_endpoint():
    data = {"key": "value"}
    return jsonify(data)

Deployment

Flask's built-in server is for development only. For production use Gunicorn:

pip install gunicorn
gunicorn app:app --bind 0.0.0.0:8000 --workers 4

One-click deploy options:

  • Railway — push the repo, set start command to gunicorn app:app.
  • Render — create a Web Service, Python env, same start command.
  • Fly.io — fly launch auto-detects Flask.
  • Google Cloud Run — add a Dockerfile with Gunicorn (see below).
# Minimal Dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 8080
CMD ["gunicorn", "app:app", "--bind", "0.0.0.0:8080", "--workers", "2"]
